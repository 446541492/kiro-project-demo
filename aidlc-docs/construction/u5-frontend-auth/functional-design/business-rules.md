# U5 业务规则 - 前端认证与个人中心

## 1. 登录规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| LG-001 | 必填字段 | 用户名和密码为必填项 |
| LG-002 | 验证码触发 | 连续 3 次登录失败后显示滑块验证码 |
| LG-003 | 2FA 流程 | 启用 2FA 的用户需要额外输入 6 位验证码 |
| LG-004 | 错误提示 | 显示后端返回的具体错误信息 |
| LG-005 | 登录跳转 | 登录成功后跳转到之前访问的页面或首页 |
| LG-006 | 失败计数 | 登录失败计数存储在 sessionStorage，关闭标签页重置 |

## 2. 注册规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| RG-001 | 用户名格式 | 3-50 字符，仅允许字母、数字、下划线 |
| RG-002 | 密码强度 | 最少 8 位，包含大写、小写、数字、特殊字符 |
| RG-003 | 密码确认 | 确认密码必须与密码一致 |
| RG-004 | 邮箱格式 | 可选，填写时需符合邮箱格式 |
| RG-005 | 手机格式 | 可选，填写时需符合手机号格式 |
| RG-006 | 滑块验证 | 注册时必须完成滑块验证 |
| RG-007 | 自动登录 | 注册成功后自动登录并跳转首页 |

## 3. 密码修改规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| PW-001 | 当前密码 | 必须输入正确的当前密码 |
| PW-002 | 新密码强度 | 与注册密码强度要求一致 |
| PW-003 | 不可相同 | 新密码不能与当前密码相同 |
| PW-004 | 成功提示 | 修改成功后显示提示，不强制重新登录 |

## 4. 两步验证规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| TF-001 | 启用流程 | 启用需要扫码 + 输入验证码确认 |
| TF-002 | 禁用验证 | 禁用需要输入当前验证码 |
| TF-003 | 恢复码展示 | 启用成功后展示恢复码，提示用户保存 |
| TF-004 | 验证码格式 | 6 位纯数字 |

## 5. 滑块验证码规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| SC-001 | 偏移精度 | 拼图偏移误差 < 5px 视为通过 |
| SC-002 | 最短时间 | 拖动时间 > 300ms（防自动化） |
| SC-003 | 轨迹检测 | 拖动轨迹需有自然抖动（非直线） |
| SC-004 | 失败重试 | 验证失败自动重新生成拼图 |
| SC-005 | Token 有效期 | captcha_token 仅单次有效 |

## 6. 表单交互规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| FM-001 | 实时验证 | 输入框失焦时触发字段验证 |
| FM-002 | 提交验证 | 提交时验证所有字段 |
| FM-003 | 加载状态 | 提交时按钮显示 loading，禁止重复提交 |
| FM-004 | 密码可见 | 密码输入框支持显示/隐藏切换 |
| FM-005 | 回车提交 | 支持回车键提交表单 |
