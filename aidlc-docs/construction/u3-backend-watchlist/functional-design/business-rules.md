# U3 业务规则 - 后端自选管理模块

## 1. 组合规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| PF-001 | 默认组合 | 注册时自动创建名为"我的自选"的默认组合 |
| PF-002 | 默认组合保护 | 默认组合不可删除 |
| PF-003 | 组合名唯一 | 同一用户下组合名称不可重复 |
| PF-004 | 组合名长度 | 组合名称 1-50 个字符 |
| PF-005 | 组合上限 | 每个用户最多创建 20 个组合 |
| PF-006 | 级联删除 | 删除组合时同时删除组合内所有标的 |
| PF-007 | 排序连续 | sort_order 从 0 开始连续递增 |
| PF-008 | 归属验证 | 所有组合操作必须验证组合属于当前用户 |

## 2. 标的规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| WI-001 | 标的唯一 | 同一组合内不可重复添加相同标的 |
| WI-002 | 标的上限 | 每个组合最多添加 200 个标的 |
| WI-003 | 批量添加 | 批量添加时跳过已存在的标的，不报错 |
| WI-004 | 批量上限 | 单次批量添加最多 50 个标的 |
| WI-005 | 行情合并 | 获取标的列表时合并实时行情数据 |
| WI-006 | 行情容错 | 行情获取失败时仍返回标的基础信息，行情字段为空 |
| WI-007 | 排序连续 | sort_order 从 0 开始连续递增 |
| WI-008 | 归属验证 | 标的操作前必须验证组合属于当前用户 |

## 3. 权限规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| AC-001 | 登录必须 | 所有自选管理接口需要 JWT 认证 |
| AC-002 | 数据隔离 | 用户只能访问自己的组合和标的 |
| AC-003 | 跨用户禁止 | 不允许操作其他用户的组合或标的 |
