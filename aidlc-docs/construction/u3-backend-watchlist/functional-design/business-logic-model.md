# U3 业务逻辑模型 - 后端自选管理模块

## 1. 组合管理流程

### 1.1 获取用户组合列表
```
请求 -> 验证用户身份
  -> 查询 user_id 对应的所有组合
  -> 按 sort_order 升序排列
  -> 附带每个组合的标的数量 (item_count)
  -> 返回组合列表
```

### 1.2 创建组合
```
请求 -> 验证用户身份
  -> 验证组合名称（非空、长度 1-50）
  -> 检查同名组合是否存在
    -> 存在: 返回 409 冲突
    -> 不存在: 计算 sort_order（当前最大值 + 1）
      -> 创建组合记录
      -> 返回组合信息
```

### 1.3 更新组合（重命名）
```
请求 -> 验证用户身份
  -> 查找组合（验证归属）
    -> 不存在或不属于当前用户: 返回 404
  -> 验证新名称（非空、长度 1-50）
  -> 检查新名称是否与其他组合重复
    -> 重复: 返回 409 冲突
    -> 不重复: 更新名称 -> 返回组合信息
```

### 1.4 删除组合
```
请求 -> 验证用户身份
  -> 查找组合（验证归属）
    -> 不存在或不属于当前用户: 返回 404
  -> 检查是否为默认组合
    -> 是默认组合: 返回 400 拒绝删除
    -> 非默认组合: 级联删除组合内所有标的
      -> 删除组合记录
      -> 返回成功
```

### 1.5 组合排序
```
请求 -> 验证用户身份
  -> 验证 ids 列表（非空、数量匹配用户组合数）
  -> 验证所有 id 均属于当前用户
    -> 不匹配: 返回 400
    -> 匹配: 按 ids 顺序更新 sort_order (0, 1, 2, ...)
      -> 返回成功
```

### 1.6 创建默认组合（注册时调用）
```
调用 -> 创建名为"我的自选"的组合
  -> is_default = True
  -> sort_order = 0
  -> 返回组合
```

## 2. 标的管理流程

### 2.1 获取组合内标的列表
```
请求 -> 验证用户身份
  -> 查找组合（验证归属）
    -> 不存在或不属于当前用户: 返回 404
  -> 查询组合内所有标的
  -> 按 sort_order 升序排列
  -> 提取所有 symbol 列表
  -> 调用 MarketService.get_batch_quotes(symbols) 获取实时行情
  -> 合并标的信息和行情数据
  -> 返回标的列表（含行情）
```

### 2.2 添加标的
```
请求 -> 验证用户身份
  -> 查找组合（验证归属）
    -> 不存在或不属于当前用户: 返回 404
  -> 检查标的是否已在组合中
    -> 已存在: 返回 409 冲突
    -> 不存在: 计算 sort_order（当前最大值 + 1）
      -> 创建标的记录
      -> 返回标的信息
```

### 2.3 批量添加标的
```
请求 -> 验证用户身份
  -> 查找组合（验证归属）
  -> 过滤已存在的标的（跳过重复，不报错）
  -> 为每个新标的计算 sort_order
  -> 批量创建标的记录
  -> 返回新增的标的列表
```

### 2.4 移除标的
```
请求 -> 验证用户身份
  -> 查找组合（验证归属）
  -> 查找标的记录
    -> 不存在: 返回 404
    -> 存在: 删除标的记录 -> 返回成功
```

### 2.5 标的排序
```
请求 -> 验证用户身份
  -> 查找组合（验证归属）
  -> 验证 ids 列表（非空、数量匹配组合内标的数）
  -> 验证所有 id 均属于该组合
    -> 不匹配: 返回 400
    -> 匹配: 按 ids 顺序更新 sort_order (0, 1, 2, ...)
      -> 返回成功
```
