# U1 业务规则 - 后端认证模块

## 1. 注册验证规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| REG-001 | 用户名格式 | 3-50 字符，仅允许字母、数字、下划线，不能以数字开头 |
| REG-002 | 用户名唯一 | 用户名在系统中必须唯一（不区分大小写） |
| REG-003 | 邮箱格式 | 必须符合标准邮箱格式（RFC 5322） |
| REG-004 | 邮箱唯一 | 邮箱在系统中必须唯一（不区分大小写） |
| REG-005 | 手机号格式 | 11 位数字，以 1 开头 |
| REG-006 | 手机号唯一 | 手机号在系统中必须唯一 |
| REG-007 | 联系方式必填 | 邮箱和手机号至少填写一个 |
| REG-008 | 密码策略 | 见密码策略规则 |

## 2. 登录验证规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| LOGIN-001 | 登录标识 | 使用用户名登录 |
| LOGIN-002 | 错误信息 | 不区分"用户名不存在"和"密码错误"，统一返回"用户名或密码错误" |
| LOGIN-003 | 验证码触发 | 连续登录失败 3 次后，后续登录必须提供验证码 |
| LOGIN-004 | 账户锁定 | 连续登录失败 5 次后，锁定账户 15 分钟 |
| LOGIN-005 | 锁定重置 | 锁定时间到期后自动解锁，失败计数归零 |
| LOGIN-006 | 成功重置 | 登录成功后，失败计数归零，清除锁定状态 |
| LOGIN-007 | 账户状态 | 被禁用的账户（is_active=False）不允许登录 |

## 3. 密码策略规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| PWD-001 | 最小长度 | 密码最少 8 个字符 |
| PWD-002 | 最大长度 | 密码最多 128 个字符 |
| PWD-003 | 大写字母 | 必须包含至少 1 个大写字母 (A-Z) |
| PWD-004 | 小写字母 | 必须包含至少 1 个小写字母 (a-z) |
| PWD-005 | 数字 | 必须包含至少 1 个数字 (0-9) |
| PWD-006 | 特殊字符 | 必须包含至少 1 个特殊字符 (!@#$%^&*()_+-=等) |
| PWD-007 | 加密方式 | 使用 bcrypt 加密，cost factor = 12 |
| PWD-008 | 修改限制 | 新密码不能与当前密码相同 |

## 4. JWT Token 规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| JWT-001 | Access Token 有效期 | 15 分钟 |
| JWT-002 | Refresh Token 有效期 | 7 天 |
| JWT-003 | Token 载荷 | 包含 user_id, username, exp, iat, token_type |
| JWT-004 | 签名算法 | HS256 |
| JWT-005 | 密钥管理 | JWT Secret 从环境变量读取，不硬编码 |
| JWT-006 | Token 刷新 | 使用 Refresh Token 获取新的 Access Token |
| JWT-007 | Refresh Token 续期 | 当 Refresh Token 剩余有效期 < 1 天时，刷新时同时返回新的 Refresh Token |
| JWT-008 | 临时 Token | 2FA 验证用临时 Token，有效期 5 分钟，token_type = "2fa_temp" |

## 5. 2FA 规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| 2FA-001 | TOTP 算法 | 基于 RFC 6238，时间步长 30 秒 |
| 2FA-002 | 时间偏差 | 允许前后 1 个时间窗口的偏差（即 ±30 秒） |
| 2FA-003 | 密钥长度 | 20 字节随机密钥，Base32 编码 |
| 2FA-004 | 二维码格式 | otpauth://totp/StocksAssist:{username}?secret={secret}&issuer=StocksAssist |
| 2FA-005 | 启用验证 | 启用 2FA 时必须先验证一次 TOTP 码，确认用户已正确配置 |
| 2FA-006 | 禁用验证 | 禁用 2FA 时必须提供当前有效的 TOTP 码 |
| 2FA-007 | 恢复码数量 | 每次生成 8 个一次性恢复码 |
| 2FA-008 | 恢复码格式 | 8 位随机字母数字组合（大写） |
| 2FA-009 | 恢复码使用 | 每个恢复码只能使用一次 |
| 2FA-010 | 恢复码重置 | 重新生成恢复码时，旧的全部失效 |
| 2FA-011 | 密钥存储 | totp_secret 加密存储在数据库中 |

## 6. 设备管理规则

| 规则编号 | 规则名称 | 规则描述 |
|---------|---------|---------|
| DEV-001 | 设备标识 | 使用浏览器指纹或 UUID 作为 device_id |
| DEV-002 | 设备名称 | 从 User-Agent 自动解析浏览器和操作系统信息 |
| DEV-003 | 设备唯一 | 同一用户的 device_id 唯一 |
| DEV-004 | 设备更新 | 已知设备登录时更新 last_login_at 和 ip_address |
| DEV-005 | 设备列表 | 按最后登录时间降序排列 |

## 7. 错误处理规则

| 错误场景 | HTTP 状态码 | 错误码 | 错误信息 |
|---------|------------|--------|---------|
| 用户名已存在 | 409 | USERNAME_EXISTS | 用户名已被注册 |
| 邮箱已存在 | 409 | EMAIL_EXISTS | 邮箱已被注册 |
| 手机号已存在 | 409 | PHONE_EXISTS | 手机号已被注册 |
| 输入格式错误 | 422 | VALIDATION_ERROR | 具体字段的验证错误信息 |
| 用户名或密码错误 | 401 | INVALID_CREDENTIALS | 用户名或密码错误 |
| 账户已禁用 | 403 | ACCOUNT_DISABLED | 账户已被禁用 |
| 账户已锁定 | 423 | ACCOUNT_LOCKED | 账户已锁定，请{minutes}分钟后重试 |
| 需要验证码 | 428 | CAPTCHA_REQUIRED | 请完成验证码验证 |
| Token 无效 | 401 | INVALID_TOKEN | Token 无效或已过期 |
| Token 过期 | 401 | TOKEN_EXPIRED | Token 已过期 |
| 需要 2FA 验证 | 200 | 2FA_REQUIRED | 需要两步验证（返回 temp_token） |
| 2FA 验证码错误 | 401 | INVALID_2FA_CODE | 验证码错误 |
| 2FA 已启用 | 409 | 2FA_ALREADY_ENABLED | 两步验证已启用 |
| 2FA 未启用 | 409 | 2FA_NOT_ENABLED | 两步验证未启用 |
| 当前密码错误 | 401 | WRONG_PASSWORD | 当前密码错误 |
| 新密码不符合策略 | 422 | WEAK_PASSWORD | 密码不符合安全要求 |
| 新旧密码相同 | 422 | SAME_PASSWORD | 新密码不能与当前密码相同 |
| 未认证 | 401 | UNAUTHORIZED | 请先登录 |
| 权限不足 | 403 | FORBIDDEN | 权限不足 |

## 8. API 请求/响应格式

### 统一错误响应格式
```json
{
  "detail": "错误描述信息",
  "code": "ERROR_CODE"
}
```

### 统一成功响应格式
```json
{
  "data": { ... },
  "message": "操作成功"
}
```

### Token 响应格式
```json
{
  "access_token": "eyJ...",
  "refresh_token": "eyJ...",
  "token_type": "bearer",
  "expires_in": 900
}
```

### 用户信息响应格式
```json
{
  "id": 1,
  "username": "testuser",
  "email": "test@example.com",
  "phone": "13800138000",
  "is_2fa_enabled": false,
  "created_at": "2026-03-05T00:00:00Z"
}
```
