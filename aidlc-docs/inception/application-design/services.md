
## 一、后端服务

### 1. 认证服务 (AuthService)

**职责**: 用户身份认证和授权管理

**编排流程**:

#### 注册流程
```
用户请求 -> 验证输入 -> 检查用户名/邮箱/手机唯一性
  -> 加密密码 -> 创建用户 -> 创建默认自选组合
  -> 生成 JWT Token -> 记录设备信息 -> 返回 Token
```

#### 登录流程
```
用户请求 -> 验证输入 -> 查找用户 -> 验证密码
  -> 检查 2FA 状态
    -> 未启用: 生成 Token -> 记录设备 -> 返回 Token
    -> 已启用: 返回 2FA 验证要求 -> 验证 TOTP -> 生成 Token -> 返回 Token
```

#### Token 刷新流程
```
Refresh Token -> 验证有效性 -> 生成新 Access Token -> 返回新 Token
```

**依赖服务**: UserService, TwoFactorService, SecurityUtils

---

### 2. 两步验证服务 (TwoFactorService)

**职责**: TOTP 两步验证的完整生命周期管理

**编排流程**:

#### 启用 2FA 流程
```
用户请求启用 -> 生成 TOTP 密钥 -> 生成二维码
  -> 返回密钥和二维码 -> 用户扫码 -> 验证首次 TOTP 码
  -> 保存密钥到数据库 -> 生成恢复码 -> 返回恢复码
```

#### 验证 2FA 流程
```
用户输入验证码 -> 获取用户 TOTP 密钥 -> 验证 TOTP 码
  -> 验证成功: 返回成功
  -> 验证失败: 检查恢复码 -> 匹配则消耗恢复码并返回成功
```

**依赖服务**: 无外部服务依赖（使用 PyOTP 库）

---

### 3. 行情服务 (MarketService)

**职责**: 行情数据获取、缓存和转换

**编排流程**:

#### 获取榜单流程
```
请求榜单 -> 检查缓存（内存缓存，30秒过期）
  -> 缓存命中: 返回缓存数据
  -> 缓存未命中: 调用 TushareClient -> 数据转换 -> 更新缓存 -> 返回数据
```

#### 搜索标的流程
```
搜索关键词 -> 判断搜索类型（代码/名称/拼音）
  -> 调用 TushareClient 搜索 -> 过滤和排序结果 -> 返回结果
```

**依赖服务**: TushareClient

---

### 4. 自选组合服务 (PortfolioService)

**职责**: 自选组合的生命周期管理

**编排流程**:

#### 创建组合流程
```
用户请求 -> 验证组合名称 -> 计算排序顺序（末尾）
  -> 创建组合记录 -> 返回组合信息
```

#### 删除组合流程
```
用户请求 -> 验证组合归属 -> 检查是否默认组合
  -> 是默认组合: 拒绝删除
  -> 非默认组合: 删除组合内所有标的 -> 删除组合 -> 重新计算排序
```

#### 排序流程
```
用户提交新排序 -> 验证所有组合 ID 归属 -> 批量更新 sort_order -> 返回成功
```

**依赖服务**: WatchlistService（删除组合时级联删除标的）

---

### 5. 自选标的服务 (WatchlistService)

**职责**: 组合内标的的管理

**编排流程**:

#### 添加标的流程
```
用户请求 -> 验证组合归属 -> 检查标的是否已存在
  -> 已存在: 返回错误
  -> 不存在: 获取标的信息 -> 计算排序顺序 -> 创建记录 -> 返回标的信息
```

#### 获取标的列表流程（含实时行情）
```
请求标的列表 -> 查询组合内标的 -> 提取所有 symbol
  -> 调用 MarketService 批量获取行情 -> 合并标的信息和行情数据 -> 返回
```

**依赖服务**: MarketService（获取实时行情）

## 二、前端服务

### 1. HTTP 客户端服务 (apiClient)

**职责**: 统一的 HTTP 请求管理

**编排流程**:

#### 请求拦截
```
发起请求 -> 检查 Token -> 注入 Authorization 头 -> 发送请求
```

#### 响应拦截
```
收到响应 -> 检查状态码
  -> 200: 返回数据
  -> 401: 尝试刷新 Token -> 成功则重试请求 -> 失败则跳转登录页
  -> 其他错误: 显示错误提示 -> 返回错误
```

### 2. 认证 API 服务 (authApi)

**职责**: 封装所有认证相关的 HTTP 请求
**交互对象**: 后端 AuthService

### 3. 行情 API 服务 (marketApi)

**职责**: 封装所有行情相关的 HTTP 请求
**交互对象**: 后端 MarketService

### 4. 自选组合 API 服务 (portfolioApi)

**职责**: 封装所有自选组合和标的相关的 HTTP 请求
**交互对象**: 后端 PortfolioService, WatchlistService

## 三、服务交互矩阵

| 调用方 | 被调用方 | 交互方式 | 说明 |
|--------|---------|---------|------|
| auth_router | AuthService | 直接调用 | 路由层调用服务层 |
| AuthService | UserService | 直接调用 | 用户查询和创建 |
| AuthService | TwoFactorService | 直接调用 | 2FA 验证 |
| AuthService | PortfolioService | 直接调用 | 注册时创建默认组合 |
| market_router | MarketService | 直接调用 | 路由层调用服务层 |
| MarketService | TushareClient | 直接调用 | 获取行情数据 |
| portfolio_router | PortfolioService | 直接调用 | 路由层调用服务层 |
| watchlist_router | WatchlistService | 直接调用 | 路由层调用服务层 |
| WatchlistService | MarketService | 直接调用 | 获取标的实时行情 |
| 前端 apiClient | 后端 API | HTTP REST | 前后端通信 |
| 前端 Store | 前端 API 服务 | 直接调用 | 状态管理调用 API |
