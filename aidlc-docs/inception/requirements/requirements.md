# 需求文档 - Stocks Assist（股票助手）

## 项目概述

**项目名称**: Stocks Assist（股票助手）

**项目类型**: 全新项目 (Greenfield)

**项目描述**: 
一个全栈 Web 应用，支持用户注册登录，并提供个性化的股票自选组合管理功能。用户可以创建多个自选组合，添加关注的股票标的，查看实时行情数据和榜单，实现个性化的投资组合管理。

---

## 意图分析

### 用户请求
开发一个系统，支持通过用户 API 进行用户注册、登录。通过行情 API 创建自选组合，为各个组合添加自选标的 Symbol。后端用户 API、行情 API 和自选标的列表 API 都已经具备。只需展示开发的前端对接 API 的效果。最终完成一个用户可以登录、注册，且根据用户个性化创建维护自选列表的功能。

### 请求类型
新项目 - 从零开始构建完整的前后端应用

### 范围估计
系统级 - 包含前端应用、后端 API、数据库、第三方 API 集成

### 复杂度估计
中等复杂度 - 涉及用户认证、数据管理、第三方 API 集成、实时数据展示

## 技术栈

### 前端技术栈
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **UI 组件库**: Ant Design 5.x
- **状态管理**: Zustand
- **路由**: React Router v6 (History 模式)
- **HTTP 客户端**: Axios
- **代码规范**: ESLint + Prettier

### 后端技术栈
- **框架**: FastAPI (Python 3.10+)
- **数据库**: SQLite
- **ORM**: SQLAlchemy
- **认证**: JWT (JSON Web Token)
- **2FA**: PyOTP (TOTP 实现)
- **行情数据**: Tushare API
- **API 文档**: 自动生成 OpenAPI/Swagger 文档

### 项目结构
- **组织方式**: Monorepo（一个仓库包含前后端）
- **目录结构**:
  ```
  stocks-assist/

  ├── backend/           # FastAPI 后端应用
  ├── aidlc-docs/        # AI-DLC 文档
  ```

## 功能需求

### 1. 用户认证与安全

#### 1.1 用户注册
- 支持手机号注册
- 支持邮箱注册
- 密码强度验证
- 图形验证码（滑块验证码）
- 注册成功后自动登录

#### 1.2 用户登录
- 支持手机号/邮箱登录
- 密码验证
- 图形验证码（滑块验证码）
- 新设备检测（记录设备信息，不做额外校验）
- JWT Token 认证
- Token 自动刷新机制

#### 1.3 两步验证 (2FA)
- 基于 TOTP 的验证器应用（Google Authenticator、Microsoft Authenticator 等）
- 生成 2FA 二维码
- 验证 TOTP 验证码
- 用户可以启用/禁用 2FA
- 备用恢复码生成

#### 1.4 用户管理
- 查看个人信息
- 修改密码
- 修改邮箱/手机号
- 登出功能
- 设备管理（查看登录设备列表）

### 2. 行情数据展示

#### 2.1 行情榜单
支持多种行情榜单类型：
- 涨幅榜（当日涨幅最大的股票）
- 跌幅榜（当日跌幅最大的股票）
- 成交量榜（成交量最大的股票）
- 热门榜（用户关注度最高的股票）
- 关注榜（平台推荐的热门股票）
- 行业板块榜（各行业板块涨跌情况）

#### 2.2 实时数据更新
- 使用轮询方式定时刷新行情数据
- 刷新间隔可配置（默认 30 秒）
- 显示最后更新时间
- 手动刷新按钮

#### 2.3 行情数据字段
根据 Tushare API 返回的字段灵活展示，基础字段包括：
- 标的代码（股票代码）
- 标的名称
- 当前价格
- 涨跌幅（百分比）
- 涨跌额
- 开盘价
- 最高价
- 最低价
- 昨收价
- 成交量
- 成交额
- 换手率
- 市盈率
- 市净率

#### 2.4 数据来源
- **Tushare API**
- 支持 A 股（沪深市场）
- 支持港股
- 支持美股
- 支持全球主要市场
- Token: 已提供（配置在环境变量中）

### 3. 标的搜索功能

#### 3.1 搜索方式
- 按标的代码搜索（如：600519）
- 按标的名称搜索（如：贵州茅台）
- 按拼音首字母搜索（如：GZMT）
- 支持模糊搜索
- 实时搜索建议

#### 3.2 搜索结果展示
- 下拉列表形式
- 显示标的代码、名称、当前价、涨跌幅
- 点击结果可快速添加到自选组合
- 支持键盘导航（上下键选择，回车确认）

### 4. 自选组合管理

#### 4.1 组合列表
- 显示用户创建的所有自选组合
- 显示组合名称
- 显示组合内标的数量
- 支持拖拽排序（调整组合显示顺序）
- 默认组合（系统自动创建，不可删除）

#### 4.2 组合操作
- **创建组合**: 输入组合名称创建新组合
- **重命名组合**: 修改组合名称
- **删除组合**: 删除组合及其内所有标的
- **切换组合**: 点击组合切换显示对应的标的列表
- **组合数量**: 无限制

#### 4.3 组合内标的管理
- 显示组合内所有标的
- 显示标的实时行情数据
- 支持拖拽排序（调整标的显示顺序）
- 标的数量无限制

#### 4.4 标的操作
- **添加标的**: 
  - 从搜索结果添加
  - 从榜单直接添加
  - 支持批量添加
- **移除标的**: 从组合中移除标的
- **移动标的**: 将标的移动到其他组合
- **查看详情**: 点击标的查看详细行情信息

### 5. 用户界面

#### 5.1 响应式设计
- 支持桌面浏览器（1920x1080、1366x768 等）
- 支持移动端浏览器（适配手机屏幕）
- 自适应布局，根据屏幕尺寸调整

#### 5.2 主题切换
- 浅色主题（默认）
- 深色主题
- 主题偏好保存到本地存储
- 跟随系统主题（可选）

#### 5.3 页面结构
- **登录/注册页**: 用户认证入口
- **首页/行情页**: 展示行情榜单和搜索功能
- **自选页**: 管理自选组合和标的
- **个人中心**: 用户信息、设置、2FA 管理
- **导航栏**: 全局导航，显示用户信息

## 数据模型

### 1. 用户表 (users)
- id: 主键
- username: 用户名（唯一）
- email: 邮箱（唯一，可选）
- phone: 手机号（唯一，可选）
- password_hash: 密码哈希
- is_2fa_enabled: 是否启用 2FA
- totp_secret: TOTP 密钥
- created_at: 创建时间
- updated_at: 更新时间

### 2. 设备表 (devices)
- id: 主键
- user_id: 用户 ID（外键）
- device_id: 设备唯一标识
- device_name: 设备名称
- last_login_at: 最后登录时间
- created_at: 创建时间

### 3. 自选组合表 (portfolios)
- id: 主键
- user_id: 用户 ID（外键）
- name: 组合名称
- sort_order: 排序顺序
- is_default: 是否默认组合
- created_at: 创建时间
- updated_at: 更新时间

### 4. 自选标的表 (watchlist_items)
- id: 主键
- portfolio_id: 组合 ID（外键）
- symbol: 标的代码
- name: 标的名称
- market: 市场类型（A股/港股/美股）
- sort_order: 排序顺序
- created_at: 添加时间

## API 接口设计

### 1. 用户认证 API

#### POST /api/auth/register
注册新用户

#### POST /api/auth/login
用户登录

#### POST /api/auth/logout
用户登出

#### POST /api/auth/refresh
刷新 Token

#### GET /api/auth/me
获取当前用户信息

### 2. 2FA API

#### POST /api/auth/2fa/enable
启用 2FA（返回二维码）

#### POST /api/auth/2fa/verify
验证 TOTP 码

#### POST /api/auth/2fa/disable
禁用 2FA

### 3. 行情 API

#### GET /api/market/rankings
获取行情榜单

#### GET /api/market/search
搜索标的

#### GET /api/market/quote/{symbol}
获取标的详细行情

### 4. 自选组合 API

#### GET /api/portfolios
获取用户所有组合

#### POST /api/portfolios
创建新组合

#### PUT /api/portfolios/{id}
更新组合信息

#### DELETE /api/portfolios/{id}
删除组合

#### PUT /api/portfolios/reorder
调整组合排序

### 5. 自选标的 API

#### GET /api/portfolios/{id}/items
获取组合内所有标的

#### POST /api/portfolios/{id}/items
添加标的到组合

#### DELETE /api/portfolios/{id}/items/{item_id}
从组合移除标的

#### PUT /api/portfolios/{id}/items/reorder
调整标的排序
