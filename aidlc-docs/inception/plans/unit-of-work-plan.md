# 单元生成计划 - Stocks Assist

## 分解策略

本项目采用 Monorepo 结构，前后端在同一仓库中。根据应用设计的组件分析，系统将分解为 4 个工作单元，按依赖顺序执行。

---

## 执行计划

### 第一部分：单元定义

- [ ] 1.1 定义单元 1：后端基础架构
  - FastAPI 项目搭建和配置
  - 数据库模型和迁移
  - JWT 认证系统
  - 用户注册/登录 API
  - 安全工具（密码加密、Token 管理）

- [ ] 1.2 定义单元 2：后端业务功能
  - 2FA (TOTP) 完整实现
  - Tushare API 客户端封装
  - 行情服务（榜单、搜索、行情查询）
  - 自选组合 CRUD API
  - 自选标的 CRUD API
  - 初始测试数据

- [ ] 1.3 定义单元 3：前端基础架构
  - React + TypeScript + Vite 项目搭建
  - Ant Design 主题配置（深色/浅色）
  - React Router 路由配置
  - Zustand 状态管理（所有 Store）
  - Axios HTTP 客户端封装（拦截器、Token 管理）
  - 全局布局组件（AppLayout、Navbar、BottomTabBar）

- [ ] 1.4 定义单元 4：前端业务功能
  - 登录/注册页面 + 滑块验证码
  - 2FA 设置和验证页面
  - 行情榜单页面
  - 标的搜索组件
  - 自选组合管理页面
  - 自选标的管理页面
  - 个人中心页面
